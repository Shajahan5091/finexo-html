Prerequisites

  • A Google Cloud Project with BigQuery API enabled
  • A google storage bucket
  • A Google Cloud Service Account
  • A Service Account Json Key to authenticate into your Service Account
  • The "roleAdmin" or "organizationRoleAdmin" roles are required to create the custom role

Creating a Service Account

It is recommended to use a dedicated service account for Snow Migrate to simplify permission management and auditing.
Here is a detailed guide that explains how to create a service account
GCP’s Guide for Creating a Service Account


Service Account Keys Configuration

Snow Migrate currently supports JSON Keys only, ensure to create your key in the same. Then, download the key as Google permits viewing its contents only at this stage.
Here is a detailed guide explaining how to create service account keys
GCP’s Guide for Creating and Managing Service Account Keys


Creating a Custom Role

Create a Custom role to add necessary permissions for Data Migration.
Steps to be followed to create the custom role for Snow Migrate:

  • Download this YAML file, which will be used in later step to create a Custom role
    • custom_role.yaml Download
  • Go to the Google Cloud Console, and then click on the Cloud Shell icon at the top right corner. This will open a terminal-like interface directly in your browser.
  • Upload the `custom_role.yaml` file to your Cloud Shell environment. You can use the upload button in the Cloud Shell toolbar to upload the file.
  • Use the `gcloud` command-line tool to create the custom role using the uploaded YAML file. Here's the command to run
    • Bash 
    
  gcloud iam roles create MigrateRole --project=`PROJECT_ID` --file=custom_role.yaml
  • Once the custom role is created, you need to assign it to the service account within your GCP project
    • Bash 
    
  gcloud projects add-iam-policy-binding YOUR_PROJECT_ID --member=serviceAccount:SERVICE_ACCOUNT_EMAIL --role=projects/YOUR_PROJECT_ID/roles/custom.MigrateRole
